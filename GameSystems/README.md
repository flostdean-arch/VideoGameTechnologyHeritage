# GameSystems

## 概要

このフォルダでは、ゲーム開発において使用されるゲームシステム層の技術についてまとめます。

CoreTechnologies がハードウェアやエンジン基盤などの低レイヤ技術を扱うのに対し、
GameSystems ではゲームプレイを構成するための汎用的な仕組みや設計パターンを扱います。

ゲーム規模の拡大に伴い、単純な処理の集合から再利用可能なシステム構造へと発展してきました。

---

## 分類

### Scene Management（シーン管理）

ゲーム内の状態や画面遷移を管理する仕組み。

主な内容：

- タイトル画面管理
- ゲーム画面切り替え
- ステージ管理
- ロード処理
- ワールド管理

---

### Entity Component System（ECS）

ゲームオブジェクトを管理するためのアーキテクチャ。

主な内容：

- Entity（実体）
- Component（属性）
- System（処理）
- データ指向設計
- ECSフレームワーク

近年のゲームエンジンでは、大規模ゲーム開発向けの設計として広く利用されています。

---

### Animation System（アニメーションシステム）

ゲーム内の動きを制御するシステム。

主な内容：

- スプライトアニメーション
- ボーンアニメーション
- キーフレームアニメーション
- アニメーションブレンド
- ステートマシン

---

### Camera System（カメラシステム）

ゲーム内の視点制御を行う仕組み。

主な内容：

- 2Dスクロールカメラ
- プレイヤー追従カメラ
- 三人称視点カメラ
- シネマティックカメラ
- カメラ演出

---

### UI System（ユーザーインターフェース）

ゲーム画面上の情報表示や操作を管理するシステム。

主な内容：

- HUD
- メニュー画面
- インベントリ
- ダイアログ
- UIフレームワーク

---

### Save System（セーブシステム）

ゲームデータを保存・復元する仕組み。

主な内容：

- セーブデータ形式
- シリアライズ
- チェックポイント
- クラウドセーブ
- データ互換性

---

### Quest / Event System（クエスト・イベントシステム）

ゲーム進行を管理する仕組み。

主な内容：

- クエスト管理
- イベントトリガー
- ミッション状態管理
- 分岐イベント
- スクリプトシステム

---

## 技術の発展

初期のゲームでは、ゲーム処理とシステム処理が密接に結合していました。

ゲーム規模が大きくなるにつれて、

- ゲーム状態管理
- オブジェクト指向設計
- コンポーネント設計
- データ駆動型設計
- ECS

などの設計手法が発展しました。

---

## 目的

ゲーム開発における主要なゲームシステム設計を記録し、
過去から現在まで利用されてきた技術や設計思想を保存することを目的とします。
