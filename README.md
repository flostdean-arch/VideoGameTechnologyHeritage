# Video Game Technology Heritage

> **失われつつあるゲーム技術を、未来へ。**

## 概要

**Video Game Technology Heritage** は、ゲームエンジンやゲームプログラミング技術の歴史を記録・整理・保存することを目的としたリポジトリです。

現在では開発が終了したゲームエンジンや、資料・開発環境の入手が困難になりつつある技術を中心に、その仕組みや歴史的背景を体系的にまとめます。

単なるゲームエンジンの紹介ではなく、

- なぜその技術が生まれたのか
- どのような課題を解決したのか
- 後世の技術へどのような影響を与えたのか
- 現在でも研究・再現できる環境は存在するのか

という「技術の継承」を重視しています。

---

# このリポジトリの目的

本リポジトリは、次の3つを目的としています。

- 歴史的なゲーム技術の記録
- 現在でも研究可能な環境の整理
- 将来へ向けた技術資料の保存

ゲーム開発技術は日々進化していますが、その一方で、古いゲームエンジンや関連資料は年々失われつつあります。

本リポジトリは、それらを「技術遺産（Technology Heritage）」として記録し、将来に残すことを目指します。

---

# 対象範囲

本リポジトリでは、**歴史的価値の高いゲーム技術**を優先して扱います。

## 優先的に扱う内容

- Unreal Engine 1〜3
- id Tech 1〜4
- Build Engine
- Doom Engine
- Quake Engine
- その他、現在では開発終了または保守が限定的なゲームエンジン

これらについては、

- 技術解説
- 歴史
- 開発背景
- 現在の研究環境
- 保存状況

まで含めて整理します。

---

## 現在も開発が続くゲームエンジンについて

Unreal Engine 4・5など、現在も公式によって継続的に開発・保守されているゲームエンジンについては、本リポジトリでは**歴史的な位置付けや技術の継承関係を説明することを主目的**とします。

最新機能の使い方や実践的な開発手法については、公式ドキュメントや専門書籍を参照してください。

# 記事の構成

ゲームエンジンの記事は、可能な限り共通の構成で整理します。

- 概要
- 開発背景
- 基本情報
- 現在の研究環境
- 保存状況
- 技術解説
- 後継技術への影響
- 採用タイトル
- 関連資料
- 出典

---

# 本リポジトリの方針

本リポジトリでは、以下を重視します。

- 一次資料・公式資料を優先する
- 推測ではなく確認可能な事実を書く
- 考察は事実と区別して記述する
- 現在でも研究できる方法を可能な限り記録する
- ライセンスや著作権を尊重する

---

# このリポジトリが目指すもの

ゲームエンジンはソフトウェアであると同時に、ゲーム開発の歴史そのものでもあります。

しかし、開発終了や配布終了によって、過去の技術へアクセスすることは年々難しくなっています。

本リポジトリは、それらを単なる懐古ではなく「技術遺産」として整理・保存し、未来の開発者や研究者が参照できる資料となることを目指しています。

---

## ライセンス・注意事項

本リポジトリは技術資料を目的としています。

掲載する情報は、公開されている資料・公式ドキュメント・ライセンスに基づき整理しています。

著作権のあるソースコードやアセットの再配布を目的とするものではありません。
