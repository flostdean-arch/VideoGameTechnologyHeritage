# Governance / 運営方針（草案）

この文書はプロジェクトの運営方針と意思決定プロセスの基本を示します。将来的に外部へ移譲・共同運営する際の基準になります。

## プロジェクトの目的
ビデオゲームに関する技術的な資料・メタデータ・保存手法を収集・整理し、未来に伝えることを目的とします。

## 役割と権限
- 管理者（Admin）
  - リポジトリ設定、権限付与、最終的な意思決定権を持ちます。
- コアメンテナー（Maintainer）
  - Issue や PR のマージ、レビュー、日常的な管理を行います。
- コントリビューター（Contributor）
  - Issue、PR、ドキュメント提供を行う一般的な参加者です。

役割は README または MAINTAINERS ファイルで明示します。

## 意思決定の流れ
- 小さな変更（ドキュメントの修正、バグ修正）はコアメンテナーの承認でマージ可能。
- 仕様変更や大きな方針変更は Issue で提案し、少なくとも 3 人の賛同（うち 1 人はコアメンテナー）を得ることを推奨。
- 意見が割れた場合はコアメンテナー会議で最終判断を行う（またはランダムに選んだメンテナーの合議制にする等、具体的なルールを詰める）。

## リリースとリポジトリ管理
- リリースは semver を推奨（可能なら自動化ツールを使う）。
- 重大な変更や履歴書き換えは事前に告知する（メーリングリスト、Issue、README の NOTICE セクションなど）。

## 貢献の受け入れ方針
- すべての貢献を歓迎しますが、法的に問題のある素材は受け入れません（違法コピー、第三者の機密資料など）。
- 大量のバイナリや大容量ファイルは別のアーカイブ手段（外部ストレージ、LFSポリシー等）で扱う。

## 移譲と終了条件
- プロジェクトを別の個人または組織に移譲する場合は、移譲条件と移譲後の主要連絡先（maintainer list）を事前に公開します。
- プロジェクト終了（アーカイブ）を決める場合も Issue による合意を前提とします。

## 紛争解決
- 個別の紛争はまず Issue 上で話し合い、それでも解決しない場合は管理者または第三者の仲裁を依頼します。
