# Contributing to VideoGameTechnologyHeritage

このプロジェクトに興味を持っていただき、ありがとうございます。

`VideoGameTechnologyHeritage` は、過去のゲーム技術を記録・整理し、
後世に残すことを目的とした技術アーカイブプロジェクトです。

ここでは「どのような情報を、どのような形で貢献してほしいか」をまとめています。

---

## 🎯 貢献で重視すること

このプロジェクトでは、単なる知識や印象ではなく、

**再現・検証・引用が可能な具体的な技術情報**

を重視しています。

後から読む人が「なぜそうだったのか」を確認できる内容が理想です。

### 良い例

* 「ファミコンのスプライトは1ライン最大8枚までしか表示できず、それを超えるとちらつきが発生した。当時のゲームではこの制約を○○という手法で回避していた」

* 「Unreal Engine 3 のレンダリングパイプラインでは○○の処理を採用しており、開発者インタビューでは△△について説明されている」

### 避けたい内容

* 「昔のゲームは容量が少なくて大変だった」
* 「昔の開発者は苦労していた」

のような、根拠や検証方法がない一般論のみの記述。

※ 確認できない情報を扱う場合は「未確認情報」と明記してください。

# 📂 情報の配置場所

基本的な分類は以下の通りです。

```
GameEngines/
    ゲームエンジン技術資料

History/
    開発者インタビュー
    年表
    業界動向
    技術史資料

Tools/
    復元プロジェクト
    非公式移植
    保存作業用ツール
```

特定エンジンの技術詳細は `GameEngines/` に配置してください。

以下の分野は将来的に追加予定です。

* `Hardware/`
* `Algorithm/`
* `Translations/`

# 📁 GameEngines の構成ルール

ゲームエンジンは、

**エンジン系統 → 世代**

の2階層で整理します。

例:

```
GameEngines/
  UnrealEngine/
    README.md
    UnrealEngine1/
      README.md
      rendering.md
      gameplay.md
    UnrealEngine2/
    UnrealEngine3/

  idTech/
    README.md
    idTech1/
    idTech2/
    idTech3/

  BuildEngine/
    README.md
```

## ルール

* 系統トップの `README.md`

  * エンジンファミリー全体の概要
  * 世代ごとの進化や違い

* 世代フォルダの `README.md`

  * その世代の概要
  * 採用タイトル
  * 技術的特徴
  * 開発環境や公開状況

* 技術詳細

  * `rendering.md`
  * `physics.md`
  * `audio.md`

など、テーマごとに分割してください。

フォルダ名にはスペースを使用しないでください。

例:

```
BuildEngine
```

# ✍️ 記事フォーマット

各記事の冒頭には以下の情報を記載してください。

```markdown
# タイトル

- 対象:
- 年代:
- 難易度:
- 出典:
```

例:

```markdown
# Unreal Engine 3 Rendering System

- 対象: Unreal Engine 3
- 年代: 2006年〜
- 難易度: 中級
- 出典: 公式資料・開発者インタビュー
```

---

# 🔍 出典について

出典は以下の優先順位を推奨します。

1. 一次資料

   * 開発者本人の発言
   * 公式技術資料
   * 当時の開発資料

2. 二次資料

   * 技術書
   * インタビュー記事
   * 信頼できる解説記事

二次資料を使用する場合は、必ずリンクまたは書誌情報を記載してください。

---

# 🚀 貢献の流れ

1. 追加したい内容を決める

2. 必要であれば Issue または Discussion で相談する

3. 該当フォルダへ Markdown ファイルを追加する

4. フォーマットに沿って記事を作成する

5. Pull Request を送信する

小さな修正や補足情報も歓迎します。

---

# 🌐 翻訳について

翻訳記事を投稿する場合は、以下を明記してください。

* 原文へのリンク
* 翻訳であること
* 著作権上の扱い

引用範囲を超える転載は行わないでください。

なお、現在このリポジトリでは日本語コンテンツの充実を優先しています。

英語版ドキュメント:

* README_EN.md
* CONTRIBUTING_EN.md

などは後日整備予定です。

# 💬 質問・相談

配置場所や書き方に迷った場合は、
Discussion または Issue で気軽に相談してください。

完璧な記事よりも、
まず記録として残すことを歓迎します。
