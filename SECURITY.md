# Security Policy / セキュリティ脆弱性対応方針

脆弱性報告やセキュリティに関する連絡方法、対応の流れを示します。

## 連絡方法（推奨）
1. GitHub のセキュリティ機能（Security Advisories / Draft）を使ってプライベートに報告してください。  
   - URL: https://github.com/flostdean-arch/VideoGameTechnologyHeritage/security/advisories (公開前に限定公開で扱えます)
2. もし GitHub の機能が使えない場合は、リポジトリ管理者のメールへ連絡してください（例: security@example.com）。※実行する場合は実メールアドレスをここに明記してください。
3. 公開 Issue で報告しないでください（脆弱性が公になると悪用のリスクが高まるため）。

## 報告に含めてほしい情報
- 発見した脆弱性の概要
- 再現手順（できるだけ具体的に）
- 期待される影響（データ漏洩、リモート実行など）
- 試した緩和策や環境（バージョン等）
- 連絡先（必要なら PGP 鍵など暗号化手段を併記）

## 対応方針とタイムライン（目安）
- 受領確認：48時間以内に受領確認を返します。
- 初動評価：受領から7日以内に脆弱性の重大度評価と暫定対応方針を提示します。
- 修正リリース：可能な限り迅速に修正を行い、セキュリティアドバイザリとして公開します（重大度とリソースに依存）。

## 開示とクレジット
- 報告者が公開を希望する場合はクレジットを掲載します（匿名希望も可）。
- 共同で脆弱性対応を行う場合、公開タイミングは修正が行われ、対策が利用可能になってからにします（Responsible Disclosure を採用）。

## 緊急対応
- サービスに重大な影響がある場合は、該当するブランチを一時的に非公開にする、または緊急パッチを優先します（該当する権限を持つ管理者が判断）。

必要ならば PGP 鍵や連絡用メールアドレスのテンプレをここに追加できます。公開連絡先を用意するか、GitHub のセキュリティアドバイザリ運用で行うのが現実的です。
