# Contributing to VideoGameTechnologyHeritage

まずはこのプロジェクトに興味を持っていただき、ありがとうございます。
ここでは「具体的に何を、どんな形で貢献してほしいか」をまとめています。

---

## 🎯 特に求めているもの

抽象的な「知識」よりも、**再現・検証・引用ができる具体性**を重視しています。
以下のような粒度で書いていただけると、後から読む人にとって価値が高くなります。

**良い例:**
- 「ファミコンのスプライトは1ライン最大8枚までしか表示できず、それを超えるとチラつく。当時のゲームはこの制約を◯◯という手法で回避していた」
- 「◯◯(ゲーム名)のカメラ制御は、プレイヤーの移動速度に応じて追従の遅延を変えるバネモデルを使っている(出典: 開発者インタビューのURL)」

**避けたいもの:**
- 「昔のゲームは容量が少なくて大変だった」のような、検証できない一般論のみの記述
- 出典が一切ない伝聞情報

## 📂 どこに書くか

- `GameEngine/<エンジン系統>/<世代>/` : ゲームエンジンの技術解説(下記参照)
- `Hardware/` : 特定のハードウェアの制約・仕組み
- `Algorithm/` : 特定のアルゴリズム・ハック技(コードサンプル歓迎)
- `History/` : インタビュー記録・年表・業界動向などの一次/二次資料
- `Translations/` : 既存記事の翻訳(原文へのリンク必須)

置き場所に迷ったら、Issueで質問してから書き始めてください。

### `GameEngine/` の構成ルール

エンジンは「系統(エンジンファミリー)」→「世代」の2階層で整理します。

```
GameEngine/
  UnrealEngine/
    README.md      ← 系統全体の概要、世代間の進化の要点
    UE1/
      README.md    ← 概要(発売年、採用タイトル、ソース公開/入手状況など)
      rendering.md ← レンダリング技術など、サブトピックごとにファイルを分ける
      gameplay.md
    UE2/
      README.md
      ...
  idTech/
    README.md      ← 系統全体の概要
    idTech1/
    idTech2/
    idTech3/
    idTech4/
```

- 系統トップの `README.md` には、その系統全体の概要と「世代ごとの差分の地図」だけを書き、個別の技術詳細は各世代フォルダに書いてください
- 世代フォルダ内は `README.md`(概要)+ サブトピックごとのファイル、という構成に揃えてください
- 新しいエンジン系統を追加する場合も、この2階層ルールに従ってください

## ✍️ 記事の最低限のフォーマット

各記事の冒頭に、以下を記載してください。

```markdown
# タイトル

- 対象: (例: ファミコン / Unreal Engine 3 / 全般)
- 年代: (例: 1985年〜)
- 難易度: 初級 / 中級 / 上級
- 出典: (書籍・インタビュー・一次資料へのリンク)
```

## 🔍 出典についてのルール

- 一次資料(開発者本人の発言、当時の技術文書)を最優先してください
- 二次資料を使う場合は、必ずリンクまたは書誌情報を明記してください
- 出典が示せない情報は「未確認情報」として明記した上で歓迎します(断定は避けてください)

## 🚀 貢献の手順

1. 追加したいトピックがあれば、まず Issue を立てて簡単に概要を書く
   (いきなり PR でも構いません)
2. `docs/` 配下の該当フォルダにファイルを作成(ファイル名は英数字・ハイフン区切り、例: `famicom-sprite-limit.md`)
3. 上記フォーマットに沿って執筆
4. Pull Request を送信

## 🌐 翻訳について

翻訳記事を投稿する場合は、必ず原文へのリンクと著作権上の扱い(引用の範囲か、許諾を得ているか)を明記してください。

## 💬 質問・相談

書き方や置き場所に迷ったときは、遠慮なく Discussion か Issue で質問してください。
完璧な記事より、まず投稿してみることを歓迎します。
